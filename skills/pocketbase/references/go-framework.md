# Go Framework Setup

PocketBase as a Go package — project init, structure, build, and core API.

---

## 1. Project Initialization

### Minimal `main.go`

```go
package main

import (
	"log"
	"os"
	"strings"

	"github.com/pocketbase/pocketbase"
	"github.com/pocketbase/pocketbase/plugins/migratecmd"

	// enable once you have at least one Go migration file
	_ "yourmodule/migrations"
)

func main() {
	app := pocketbase.New()

	// loosely check if it was executed using "go run"
	isGoRun := strings.HasPrefix(os.Args[0], os.TempDir())

	migratecmd.MustRegister(app, app.RootCmd, migratecmd.Config{
		// auto-create migration files on Dashboard changes (dev only)
		Automigrate: isGoRun,
	})

	if err := app.Start(); err != nil {
		log.Fatal(err)
	}
}
```

### Setup Commands

```bash
# init module
go mod init yourmodule
go get github.com/pocketbase/pocketbase

# create migrations directory
mkdir -p migrations
```

> **Replace `yourmodule`** with your actual module path (e.g., `github.com/user/myapp`). The blank import `_ "yourmodule/migrations"` must match.

## 2. Recommended Directory Structure

```
project/
├── main.go               # app entry point
├── go.mod / go.sum
├── migrations/            # Go migration files
├── hooks/                 # custom hooks & routes (optional, organize as you like)
├── pb_data/               # SQLite data — add to .gitignore
├── pb_migrations/         # auto-generated migration files (Go or JS)
└── .env                   # PB_URL, PB_SUPERUSER_EMAIL, PB_SUPERUSER_PASSWORD
```

- `migrations/` — hand-written Go migrations registered via `m.Register()`.
- `pb_migrations/` — auto-generated by `migratecmd` when Dashboard changes are made (with `Automigrate: true`).
- Both directories are loaded automatically if the blank import and `migratecmd.MustRegister()` are in place.

## 3. Build and Run

| Task | Command |
|------|---------|
| Dev run | `go run . serve` |
| Production build | `go build -o myapp . && ./myapp serve` |
| Create superuser | `go run . superuser create admin@example.com password` |
| Background (Claude Code) | `nohup go run . serve --http=127.0.0.1:8090 > pb.log 2>&1 &` |

> Use `nohup` when running from Claude Code — plain `&` dies when the shell session ends.

After starting, verify with:

```bash
python .claude/skills/pocketbase/scripts/pb_health.py
```

---

## 4. Coexistence with Standalone Features

| Feature | Behavior in Go package mode |
|---------|-----------------------------|
| `pb_hooks/*.pb.js` (JSVM) | Still loaded — but mixing Go hooks and JSVM hooks on the same event is not recommended |
| `pb_migrations/*.js` | Still auto-loaded at startup alongside Go migrations |
| REST API | Identical — all Python scripts (`pb_collections.py`, `pb_records.py`, etc.) work as-is |
| Admin Dashboard | Available at `/_/` as usual |
| E2E tests | No changes needed — tests use REST API |

**Recommendation:** Pick one language for hooks and migrations (Go or JSVM), don't mix.

## 5. `app` Object — Key Methods

### Record Queries

```go
// find by ID
record, err := app.FindRecordById("posts", recordId)

// find first matching filter
record, err := app.FindFirstRecordByFilter("posts", "slug={:slug}", dbx.Params{"slug": slug})

// find multiple with filter, sort, limit, offset
records, err := app.FindRecordsByFilter("posts", "status='published'", "-created", 20, 0)

// find all (no filter)
records, err := app.FindAllRecords("posts")
```

### Record Mutation

```go
collection, err := app.FindCollectionByNameOrId("posts")

// create
record := core.NewRecord(collection)
record.Set("title", "Hello")
err = app.Save(record)

// update
record, err := app.FindRecordById("posts", id)
record.Set("title", "Updated")
err = app.Save(record)

// delete
err = app.Delete(record)
```

### Auth Helpers

```go
authRecord, err := app.FindAuthRecordByEmail("users", "user@example.com")
```

### Raw SQL

```go
// execute (no result)
_, err := app.DB().NewQuery("DELETE FROM posts WHERE status = 'archived'").Execute()

// query single row
type Result struct {
    Count int `db:"count"`
}
result := Result{}
app.DB().NewQuery("SELECT COUNT(*) as count FROM posts").One(&result)

// query multiple rows
results := []Result{}
app.DB().NewQuery("SELECT COUNT(*) as count FROM posts GROUP BY status").All(&results)
```

### Collection Lookup

```go
collection, err := app.FindCollectionByNameOrId("posts")
```

### Typed Record Getters

```go
record.GetString("title")
record.GetInt("count")
record.GetBool("active")
record.GetStringSlice("tags")    // multi-select / multi-relation
record.GetDateTime("created")
record.Get("data")               // returns any (for JSON fields)
```
